## 1. SphereWidget — Orange JARVIS Morphing Blob

### Shape

Replace all circle-based drawing with a 24-point morphing polygon. Each point's radius:

```
r_i = base_r
    + base_r * 0.18 * sin(angle_i * 3 + t * 0.05)
    + base_r * 0.12 * sin(angle_i * 5 - t * 0.03)
    + base_r * 0.08 * cos(angle_i * 7 + t * 0.07)
```

`smooth=True` on `create_polygon` makes it flow like liquid. Three rendering layers (glow outer → glow mid → core) use the same distortion at 160% / 130% / 100% radius, each blended toward `BG` (#060810) to create a soft halo.

### Colors

| State | Core fill | Outline / glow | Morph speed |
|-------|-----------|----------------|-------------|
| Idle | `#3a1800` | `#ff8c00` | t += 1/frame (slow) |
| Speaking | `#5a2500` | `#ffaa00` | t += 2/frame (fast) |
| Muted | `#0a0015` | `#6a00ff` | frozen (t paused) |

### Glow color blending

Each glow layer blends the orb color toward BG (`#060810` → r=6, g=8, b=16). For a layer at `alpha` (0.0–1.0):
```python
r = int(BG_r + (orb_r - BG_r) * alpha)
g = int(BG_g + (orb_g - BG_g) * alpha)
b = int(BG_b + (orb_b - BG_b) * alpha)
```
Outer layer alpha=0.25, mid alpha=0.5.

### Speaking rings

Three oval outlines (not polygons) animate outward from blob center when speaking. Same logic as current implementation, re-colored to orange (`#ff8c00` → transparent).

### Canvas sizing

- `H = 160` (raised from 130)
- Canvas packs with `fill="x"` — `cx = self.winfo_width() // 2` computed each draw call so blob is always centered regardless of column width.
- `base_r = 38` idle, `42` speaking, `30` muted.

### Separator removal

Delete the `tk.Frame(center, bg=DIM, height=1).pack(fill="x")` line between sphere and chat (`_build_ui`, currently line ~2198).

## 3. Camera Feed (replaces Radar)

### Dependencies

- `opencv-python` (`cv2`) — `pip install opencv-python`
- `Pillow` (`PIL`) — likely already installed; `pip install Pillow` if not

### Widget

`tk.Canvas` sized 130×90px, replacing the current 90×90 radar canvas. Label changed from `"RADAR"` to `"CAMERA"` in `HC` color.

### Camera thread

`_start_camera()` spawns a daemon thread:

```
cap = cv2.VideoCapture(0)
if cap.isOpened():
    loop:
        ret, frame = cap.read()
        if ret:
            frame_rgb = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
            img = Image.fromarray(frame_rgb).resize((130, 90))
            img_tk = ImageTk.PhotoImage(img)
            self.after(0, self._show_frame, img_tk)
        sleep(0.033)
else:
    self._cam_available = False
```

`_show_frame(img_tk)` stores reference (prevent GC) and draws on canvas.

### NO SIGNAL animation

When `self._cam_available` is False (or `cv2` import fails), `_animate_camera()` runs on the 33ms loop:

- Dark canvas (`HBG` fill)
- Horizontal scan-lines: 5 lines, y positions = `(t * 2 + i * 18) % 90` — scrolls downward
- `NO SIGNAL` text centered, color `#004840`
- Blinking `●` top-right corner: visible when `t % 30 < 15`, color `#ff4444`
- If cv2 not installed: additional static line `"(install opencv-python)"` in small font

### Graceful degradation

The entire cv2 import is wrapped in `try/except ImportError`. If missing, `_cam_available` is set to False immediately and the NO SIGNAL animation runs permanently.

### Cleanup

`cap.release()` called in the thread's `finally` block.

## 4. Self Repair Function

### HUD placement

Below the camera canvas, after a divider. Two sub-components:

1. **Header row**: `tk.Frame` with `⚕ SELF REPAIR` label (left) + `[RUN]` button (right). Button triggers `_run_self_repair()` in a daemon thread.
2. **Output area**: `tk.Text` widget, height=6 lines (~100px), `state="disabled"`, `bg="#020808"`, `fg=HC`. Color tags: `ok` = `#00d4aa`, `fail` = `#ff4444`, `warn` = `#ffaa00`.

### Diagnostics (run in thread, update UI via `self.after(0, ...)`)

Results are appended line-by-line using `text.configure(state="normal") → insert → configure(state="disabled")`.

1. **API key**: `get_api_key()` — PASS if non-empty, FAIL otherwise
2. **Network**: `requests.head("https://api.anthropic.com", timeout=3)` — PASS/FAIL
3. **Tool load**: for each `*.py` in `tools/custom/`, attempt `importlib.util.spec_from_file_location` + `exec_module`. Report filename + error for any that fail.
4. **Ollama**: `OllamaCore().ollama_available()` — ONLINE/OFFLINE
5. **Packages**: try importing `cv2`, `sounddevice`, `edge_tts`, `psutil`, `PIL` — list missing
6. **Error log**: read last 100 lines of `hubert_errors.log`, count entries timestamped within 24h, show count + 3 most recent error summaries (first line only)

### HUBERT tool

`tools/custom/self_repair.py` — runs the same 6 checks and returns a formatted string. No UI dependency. `run(params)` accepts optional `{"verbose": true}` to include full error log snippets.

### Error routing (replaces chat error banner)

The `HubertApp._error()` method currently displays errors as red messages in `ChatDisplay`. Change it to route all errors into the Self Repair output area instead of the chat:

- `_error(msg)` no longer calls `self.chat.error(msg)`. Instead it calls `self.hud_panel.log_error(msg)`.
- `HUDPanel.log_error(msg)` appends the message to the Self Repair text area with a `fail` color tag and a timestamp. The Self Repair section is always visible so the user can see errors without them polluting the conversation.
- The `_set_status("error")` dot indicator still updates so the user knows something went wrong.
- `on_status` (rate limit retry messages) remains in the chat as a grey italic line — these are informational, not errors.
