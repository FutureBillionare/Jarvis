# HUBERT Second Brain — Design Spec
*2026-04-12*

## Goal
Transform HUBERT's Obsidian vault into a fully automatic "second brain" that:
- Self-organizes via plugins and nightly automation
- Shares active memory between HUBERT and all swarm agents
- Gives Jake a live, always-current visual knowledge graph

---

## Vault Structure

```
HUBERT_Vault/
├── 00 - Index.md                  ← master dashboard (Dataview queries)
├── HUBERT_Memory_Map.canvas       ← auto-rebuilt nightly + after extraction
│
├── Sessions/                      ← one note per conversation
├── Daily/                         ← daily digest notes (Periodic Notes)
├── Weekly/                        ← weekly rollup (auto-generated by HUBERT)
│
├── Memory/
│   ├── People/
│   ├── Projects/
│   ├── Decisions/
│   ├── Action Items/
│   ├── Facts/
│   ├── Concepts/
│   └── Insights/
│
├── Swarm/
│   ├── _active/
│   │   ├── context.md             ← current task context (all agents read/write)
│   │   ├── working_memory.md      ← scratch space for agent findings
│   │   └── agent_states.md        ← which agents are active and what they're doing
│   ├── _completed/                ← archived research outputs
│   └── {agent-name}/              ← per-agent dedicated notes
│
├── System/
├── HUBERT Dreams/
└── .obsidian/
    └── plugins/                   ← 8 community plugins
```

Auto Note Mover routes new notes to the correct folder based on `type:` frontmatter field.

## Frontmatter Schema

### Universal fields (every note)
```yaml
---
id: "20260412-session-001"
type: "session"           # session | decision | action-item | fact | concept | person | project | insight
created: 2026-04-12
modified: 2026-04-12
status: "active"          # active | archived | completed | pending
tags: [sessions, hubert]
author: "hubert"          # hubert | gemma-swarm | haiku-swarm | claude
---
```

### Type-specific fields
| Type | Extra fields |
|------|-------------|
| `session` | `session_topic`, `duration_minutes`, `key_outcomes` |
| `decision` | `decision_owner`, `decision_status`, `confidence_level` |
| `action-item` | `assigned_to`, `due_date`, `priority`, `blocking` |
| `fact` | `confidence`, `evidence_strength`, `reviewed` |
| `person` | `person_role`, `person_expertise`, `person_projects` |
| `project` | `project_status`, `project_owner`, `project_deadline` |
| `insight` | `confidence`, `evidence_strength` |

### Relational fields (flat wikilink arrays — fastest for Dataview)
```yaml
person_refs: ["[[Jake]]"]
project_refs: ["[[HUBERT_AGI_Bot]]"]
related_to: ["[[Some Other Note]]"]
depends_on: []
```

**Rules:**
- Flat arrays only — no nested objects
- Wikilinks for cross-refs, not raw IDs
- `lower_snake_case` field names throughout
- YAML frontmatter only (no inline fields)

## Nightly Automation Pipeline

Runs at 2 AM via `memory_pipeline.py`, triggered by dream scheduler in `main.py`.

```
Session ends
    └── HUBERT writes Session note → Auto Note Mover routes to Sessions/

Nightly at 2 AM:
    ├── 1. EXTRACTION PASS (Claude)
    │       Read all new session notes since last run
    │       Extract: people, decisions, action items, facts, insights
    │       Write typed notes → Auto Note Mover routes each to correct folder
    │       Update person_refs / project_refs on source session note
    │       → Trigger canvas rebuild immediately after
    │
    ├── 2. SWARM SYNC
    │       Read Swarm/_active/working_memory.md
    │       Merge findings into relevant Memory/ notes
    │       Archive completed outputs to Swarm/_completed/
    │       Reset working_memory.md for next day
    │
    ├── 3. WEEKLY ROLLUP (Sundays only)
    │       Claude summarizes week's sessions into Weekly/ note
    │       Highlights: decisions made, actions opened/closed, people mentioned
    │
    └── 4. CANVAS REBUILD
            Nodes = all active projects + people + recent sessions + new entities
            Edges = derived from ALL frontmatter wikilinks
            Colors by type: session=purple, person=blue, project=green,
                            decision=yellow, action-item=red, insight=teal
            Saves to HUBERT_Memory_Map.canvas
            (also runs immediately after step 1)
```

### Mid-session canvas refresh
After HUBERT writes any significant new note (project, person, decision), it queues an immediate canvas refresh so the map is always current.

## New Python Module: `memory_pipeline.py`

Responsibilities:
- `extract_entities(session_path)` — Claude call to extract typed entities from a session note
- `write_typed_note(entity)` — write note with correct frontmatter to vault inbox
- `sync_swarm_memory()` — merge working_memory.md into vault
- `build_weekly_rollup()` — Claude summarizes week (Sundays only)
- `rebuild_canvas()` — reads all active notes, generates canvas JSON
- `run_nightly()` — orchestrates all of the above

Called from `main.py`'s `_start_dream_scheduler()` at 2 AM alongside Dream Mode.

## Swarm Shared Memory Protocol

- All agents (HUBERT, Gemma swarm, Haiku, Claude) read `Swarm/_active/context.md` at task start
- Agents write findings to `Swarm/_active/working_memory.md`
- `agent_states.md` tracks who is active and what they're working on
- Nightly sync merges working_memory into permanent Memory/ notes
- HUBERT's `chat()` loop prepends active context from `context.md` to every message
